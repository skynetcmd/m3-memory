## Cross-cutting reports

### `ENV_VAR_RECONCILE_REPORT.md` (2026-05-14)

Inventories **108 environment variables** across the m3-memory surface:
- **73** `M3_*`-prefixed vars in active use
- **32** non-prefixed vars (recommended for `M3_*` namespacing under deprecation alias)
- **3** auth/credential vars (intentionally unprefixed; touch FIPS path)

Produced as input to Section 9.6 of the Project "Oxidation" Rust transition plan (`~/m3_oxidation_plan.md`). The Rust binding crate (`m3-core-py`) must surface all 73 existing `M3_*` vars unchanged for backward compatibility, with typed configs derived from them.

Cross-checked against [INDEX.md](INDEX.md) — **no drift** between the tool inventory and the env-var reader set.

## Regenerating the per-tool docs

The auto-generated `<tool_name>.md` files and `INDEX.md` are produced by:

```
python bin/gen_tool_inventory.py
```

Run this after changing any tool. Entries in `INDEX.md` whose `sha1` no longer matches the live file need re-validation.

## Regenerating the env-var reconcile report

The env-var report is not auto-regenerated (the search patterns are heuristic and need a human-in-the-loop review for new patterns). To refresh it:

1. Re-run the search-pattern sweep documented in the report's "Methodology" section.
2. Cross-check every reader file path against the current [INDEX.md](INDEX.md).
3. Update the conflicts/gotchas section if any new collisions surface.

Trigger conditions for regeneration:
- New env vars added to any tool
- [INDEX.md](INDEX.md) regenerated and adds/removes tools
- A new oxidation-plan phase introduces additional `M3_*` vars
